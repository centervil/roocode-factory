# Development Log: Issue #44 - Roo-Orchestrator インフラ基盤の構築

## 日時
2026年2月28日

## 作業概要
Gemini CLI から Roo Code を遠隔操作するための Docker コンテナ環境を構築した。
`code-server` をベースに、`Tailscale` によるセキュアな VPN 接続と、`Playwright` によるブラウザ操作を可能にする構成を実現。

## 変更内容
- `.infra/` ディレクトリの作成
- `Dockerfile`: `codercom/code-server:latest` をベースに Node.js, Tailscale, Playwright をインストール。
- `docker-compose.yml`: 永続化ボリューム、ネットワーク権限（NET_ADMIN等）を設定。
- `scripts/entrypoint.sh`: Tailscale デーモンと code-server の同時起動、パーミッション修正。
- `scripts/test_playwright.js`: Playwright が code-server UI にアクセス可能かを確認するテストスクリプト。
- `gh CLI`: Dockerfile にインストール処理を追加し、`gh-config-data` ボリュームで認証情報を永続化。
- `.infra/.env.example`: 環境変数のテンプレート。

## 検証結果
- `docker-compose up -d --build` による正常起動を確認。
- `docker logs` にて、Tailscale と code-server の起動ログを確認。
- `docker exec` 内で `test_playwright.js` を実行し、`code-server login` 画面の DOM を取得できることを確認。
- `tailscale status` にて、Tailscale デーモンが正常に稼働していることを確認（未認証状態）。

## 課題・留意点
- `code-server` のデフォルトユーザーは `coder` (UID 1000) であるため、マウントされるボリュームのパーミッション設定に留意した（`entrypoint.sh` 内で `chown` を実行）。
- `Tailscale` の認証を完了するには、初回起動時に `docker exec roo-orchestrator tailscale up` を実行するか、`TAILSCALE_AUTH_KEY` を環境変数で渡す必要がある。
